## 🔧 操作指南

### 启动新任务
```bash
# 1. 读取 STATE.json
cat skills/voice-system/STATE.json

# 2. 找到 nextAction 字段
# 3. 执行对应任务
# 4. 更新 STATE.json：
#    - 移动任务：pending → inProgress
#    - 更新 nextAction
#    - 添加 note
```

### 完成任务
```bash
# 1. 更新 STATE.json：
#    - 移动任务：inProgress → completed
#    - 添加 completedAt 时间戳
#    - 更新 metrics
# 2. 更新 TASK-QUEUE.md：
#    - 移动任务到"已完成"区域
#    - 更新进度统计
```

### 遇到阻碍
```bash
# 1. 更新 STATE.json：
#    - status: running → blocked
#    - blocker.issue: 描述问题
#    - blocker.attempts: 记录尝试
# 2. 尝试解决（最多 3 次）
# 3. 3 次失败后：求助人类或其他特工
```

## 💡 核心原则

1. **状态文件是指令，不是建议**
   - 必须严格遵守 STATE.json 中的状态
   - 不要自行决定跳过或修改优先级

2. **Read state → Act → Update state**
   - 每次行动前：读状态
   - 执行任务：专注完成
   - 完成后：立即更新状态

3. **不要等人类提醒**
   - 主动读取状态文件
   - 自主推进工作
   - 受阻 3 次后才求助

4. **随时中断/恢复**
   - 状态文件保存完整上下文
   - 中断后可以从状态文件恢复
   - 不丢失进度
