# 能力对比：OpenClaw vs Airi

> 分析时间：2026-03-06 14:20
> 分析者：xiaoxiaohuang 🐤

---

## 📊 核心定位对比

| 维度 | Airi | OpenClaw (我) |
|------|------|--------------|
| **定位** | 虚拟主播/数字宠物 | AI 助手/Agent |
| **目标** | 游戏 + 直播 + 陪伴 | 任务执行 + 自动化 |
| **形态** | Live2D/VRM 虚拟形象 | 文本/语音对话 |
| **运行** | 浏览器/桌面/手机 | 任意平台 (Node.js) |

---

## 🧠 核心能力对比

### 1. 大脑 (LLM 集成)

| 能力 | Airi | OpenClaw | 差距 |
|------|------|----------|------|
| LLM 支持 | 15+ 提供商 | 10+ 提供商 | ✅ 持平 |
| 本地推理 | WebGPU 本地运行 | 依赖 API | ⚠️ 落后 |
| 角色扮演 | 深度定制 | 基础支持 | ⚠️ 落后 |

**Airi 优势**:
- 本地 WebGPU 推理（无需 API）
- 深度角色扮演系统
- 专为虚拟主播优化

**OpenClaw 优势**:
- 更简单的 API 集成
- 专注任务执行
- 更轻量

---

### 2. 记忆系统

| 能力 | Airi | OpenClaw | 差距 |
|------|------|----------|------|
| 数据库 | DuckDB WASM / pglite | duckdb-memory (模拟) | ⚠️ 落后 |
| RAG | Memory Alaya (WIP) | 基础 memory_search | ⚠️ 落后 |
| 向量搜索 | WebGPU 本地 | 待实现 | ❌ 缺失 |
| 批量搜索 | ✅ (已整合) | ✅ (已整合) | ✅ 持平 |

**Airi 优势**:
- 真实 DuckDB WASM 集成
- 向量相似度搜索
- 完整的 RAG 架构

**OpenClaw 优势**:
- stream-queue 批量处理
- 更简单的 API

**差距分析**:
- 缺少真实 WASM 加载（当前是模拟模式）
- 缺少向量嵌入和相似度搜索
- 缺少长期记忆管理架构

---

### 3. 语音能力

| 能力 | Airi | OpenClaw | 差距 |
|------|------|----------|------|
| ASR | 客户端语音识别 | 依赖火山引擎 | ⚠️ 落后 |
| TTS | ElevenLabs | 火山引擎 | ✅ 持平 |
| 队列管理 | ❌ | ✅ stream-queue | ✅ 领先 |
| 说话检测 | ✅ | ❌ | ❌ 缺失 |

**Airi 优势**:
- 多种 TTS 提供商
- 说话检测（打断支持）
- Discord 语音集成

**OpenClaw 优势**:
- TTS 队列管理（批量处理）
- 错误隔离

**差距分析**:
- 缺少本地 ASR（Whisper 等有集成但未完善）
- 缺少实时语音打断
- 缺少 Discord 语音通道集成

## 🎯 我的核心优势

### 1. Agent 任务执行 ⭐⭐⭐⭐⭐
```typescript
// 子代理分发
const result = await sessions_spawn({
  task: '研究 X 项目',
  mode: 'run',
})

// 任务队列管理
const queue = createQueue({ handlers: [...] })
queue.enqueue({ task: 'something' })
```

### 2. 文件/系统操作 ⭐⭐⭐⭐⭐
```typescript
// 文件读写
await write('file.txt', 'content')
const content = await read('file.txt')

// 系统命令
await exec('git status')
```

### 3. 浏览器自动化 ⭐⭐⭐⭐⭐
```typescript
// Playwright 控制
browser.open(url)
browser.screenshot(targetId)
browser.act(targetId, kind: 'evaluate', fn: JS)
```

### 4. 批量处理 ⭐⭐⭐⭐⭐
```typescript
// 批量记忆搜索
const results = await searchQueue.searchBatch([
  { query: '待办事项' },
  { query: '项目进度' },
])

// 批量 TTS
await Promise.all([
  tts.synthesize({ text: '第一条' }),
  tts.synthesize({ text: '第二条' }),
])
```

## 🚀 可追赶的领域

### 优先级 1: 本地向量搜索 ⭐⭐⭐⭐⭐

**当前**: 关键词搜索 (`LIKE '%query%'`)
**目标**: 向量相似度搜索

**需要**:
1. 集成 embedding 模型（本地或 API）
2. 向量存储（DuckDB 支持）
3. 相似度计算

**预计工作量**: 2-3 小时

---

### 优先级 2: 真实 DuckDB WASM ⭐⭐⭐⭐

**当前**: 模拟模式
**目标**: 真实 WASM 加载

**需要**:
1. 加载 `@duckdb/duckdb-wasm`
2. 配置 WASM 文件路径
3. 测试浏览器/Node.js 兼容

**预计工作量**: 1 小时

---

**当前**: 依赖火山引擎 API
**目标**: 本地 Whisper 运行

**需要**:

2. 优化性能和准确率
3. 流式支持

**预计工作量**: 1-2 小时
