## 🔄 执行流程

### 阶段 1: 任务启动 (5 分钟)

1. 创建 4 个文件
2. 填写初始状态
3. 列出任务队列
4. 记录开始时间

### 阶段 2: 任务执行 (循环)

```
while (任务未完成) {
  1. 读取状态文件，知道当前任务
  2. 执行子任务
  3. 更新状态文件
  4. 记录到开发日志
  5. 更新任务队列
  6. 检查上下文，必要时写文件
}
```

### 阶段 3: 任务完成 (10 分钟)

1. 写总结报告
2. 更新 lessons.md
3. 归档文件
4. 清理临时文件

---

## 📏 最佳实践

### 1. 更新频率

| 文件 | 频率 |
|------|------|
| 状态文件 | 每完成一个子任务 |
| 任务队列 | 发现新任务时 |
| 开发日志 | 每个重要操作后 |
| 总结报告 | 任务完成后 |

### 2. 上下文管理

| 使用率 | 行动 |
|--------|------|
| < 50% | 正常对话 |
| 50-70% | 开始写关键决策 |
| 70-85% | 主动写完整报告 |
| > 85% | 紧急写入所有上下文 |

### 3. 任务分解

```
大任务 → 中任务 → 小任务 → 可执行步骤

例如:
研究 Airi
  ↓
克隆 + 部署 + 分析 + 整合
  ↓
克隆仓库 (5 分钟)
安装依赖 (10 分钟)
分析架构 (15 分钟)
提取模块 (10 分钟)
写总结 (10 分钟)
```

## 💡 核心原则

1. **文件是外置大脑** - 不要依赖记忆
2. **状态驱动** - 不是时间驱动
3. **早写比晚写好** - 立即记录
4. **结构化胜过完整** - 4 个小文件 > 1 个大文件
