# 任务执行策略

> 基于 CosyVoice 3.0 安装任务经验 (2026-03-07)

---

## 📊 状态驱动流程

### 任务开始
```
1. 理解任务 → 确认目标、约束、期望
2. 评估复杂度 → 简单直接做 vs 复杂先规划
3. 创建文档 → tasks/任务名.md (复杂任务)
4. 决定团队 → 单代理 vs 智能体团队
```

### 任务执行
```
1. 定期同步 → 每 30 分钟进度更新
2. 关键节点 → 立即通知用户
3. 遇到问题 → 2 次失败后汇报
4. 智能体 → 协调子代理工作
```

### 任务完成
```
1. 交付结果
2. 记录经验 → memory/日期.md
3. 更新状态 → tasks/todo.md
4. 总结方法 → 可复用文档
```

---

## 🤖 智能体团队策略

### 适用场景
| 场景 | 推荐 | 说明 |
|------|------|------|
| 多步骤安装 | ✅ 强烈推荐 | 并行处理，效率 3 倍 |
| 研究探索 | ✅ 推荐 | 分工调研不同方案 |
| 数据处理 | ✅ 推荐 | 并行处理数据块 |
| 简单查询 | ❌ 不推荐 | 单代理更高效 |
| 日常对话 | ❌ 不推荐 | 不需要子代理 |

### 团队架构
```
主代理 (协调者)
├─ 下载代理 (负责下载任务)
├─ 安装代理 (负责依赖安装)
├─ 测试代理 (负责验证)
└─ 文档代理 (负责记录)
```

### 子代理管理
1. **创建时** - 明确任务目标和预期
2. **执行中** - 定期收集进度
3. **汇总后** - 同步给用户
4. **遇到问题** - 及时介入解决

## 📝 记忆记录策略

### 文件层级
| 文件 | 用途 | 更新频率 |
|------|------|----------|
| `memory/日期.md` | 日常记录 | 每天 |
| `MEMORY.md` | 长期记忆 | 重要事件 |
| `tasks/todo.md` | 任务追踪 | 实时更新 |
| `tasks/策略.md` | 方法总结 | 任务完成后 |

### 记录原则
> "心里记住"不靠谱，写下来才持久！

### 记录时机
- 任务开始 → 创建任务文档
- 关键决策 → 记录原因和方案
- 用户偏好 → 立即写入 MEMORY.md
- 任务完成 → 总结经验教训

---

## ⚠️ 异常处理策略

### 失败处理
| 失败次数 | 行动 |
|----------|------|
| 第 1 次 | 自己尝试解决 |
| 第 2 次 | 换方案继续试 |
| 第 3 次 | 汇报用户，请求指导 |

### 沟通时机
1. 需求不明确 → 问清楚再做
2. 有多个方案 → 列出优劣让用户选
3. 遇到技术难点 → 说明情况，给建议
4. 需要外部资源 → 如发邮件、注册账号
5. 可能失败的风险 → 提前告知

---

## 🎯 核心原则

```
✅ 能自己解决就直接干
❓ 不确定就先问清楚
📝 做完一定记录经验
🔄 遇到问题灵活调整
💡 优先用最高效的方式
🤖 复杂任务用智能体团队
```
